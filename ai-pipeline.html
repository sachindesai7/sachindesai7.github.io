Local AI · 3D Research

ComfyUI + Trellis 2.0

Explored running a fully local AI 3D generation pipeline — no cloud, no API costs, no data leaving the studio. Using Microsoft Research's Trellis 2.0 image-to-3D model inside a ComfyUI node graph, a single reference image is converted into a detailed 3D mesh with PBR textures in minutes. The result is then brought into Blender to compare the raw AI mesh against a manually retopologised version — measuring the real gap between what the model gives you and what production actually needs.

3D Generation — Trellis 2.0 (Microsoft Research)  ·  ComfyUI    Cleanup — Blender  ·  Manual Retopo

01 — Image to 3D with ComfyUI + Trellis

  • Trellis 2.0 runs entirely locally on GPU — zero per-generation cost once set up
  • ComfyUI node graph controls the full pipeline: image input → mesh → texture → export
  • Generates high-detail source mesh with PBR textures from a single reference image
  • Output in minutes vs. hours for a hand-modelled equivalent base mesh
  • No internet connection required — suitable for NDA-protected studio assets

02 — Retopo vs Original · Blender Comparison

  • Raw AI mesh: 80K–150K polys, no edge-flow, not deformation-safe
  • Retopologised mesh: ~8K polys, clean quads, animation-ready edge loops
  • AI handles the sculpt and surface detail — human handles the craft layer
  • Hybrid saves ~35–40% compared to building the full character from scratch
  • Validates the hybrid pipeline as viable for game-ready character production
3D Model Testing · 3daistudio.com

Online AI 3D Model Generation

Tested the full image-to-3D-to-rig workflow using 3daistudio.com — a single platform that runs multiple AI model generators (Tripo, Meshy and others) side by side, letting you compare results before committing to one. The workflow starts with AI-generated reference images and ends with a rigged, posable character — entirely inside online tools, no local GPU required.

Workflow: Generate front + side view images in ChatGPT or Gemini → feed both views into 3daistudio.com → run Tripo and Meshy in parallel and pick the best result → auto-rig the chosen mesh inside the same platform → export game-ready.

Images — ChatGPT  ·  Gemini    3D + Rigging — Tripo  ·  Meshy  ·  3daistudio.com

01 — AI Reference Images (Front + Side)

  • Character designed entirely using ChatGPT / Gemini image generation
  • Front view and side view created separately to give the 3D model generator full spatial data
  • Two-view input dramatically improves mesh accuracy over single-image generation
  • No concept artist involved — full character sheet produced in under 30 minutes

02 — 3D Model Generation · Tripo vs Meshy

  • Both Tripo and Meshy run simultaneously from the same reference — pick the best output
  • Full PBR textured mesh generated in 2–4 minutes per model
  • 3daistudio.com comparison view lets you judge topology, surface detail and texture quality side by side
  • Props and environment assets reach near-game-ready quality; characters need retopo
  • No local GPU needed — runs fully in the browser on any machine

03 — Auto-Rigging the AI Mesh

  • Auto-rig applied inside 3daistudio.com — no external DCC needed for initial pose testing
  • Humanoid skeleton mapped automatically to the AI-generated mesh
  • Good enough for previs, pitch videos and prototype gameplay — not production-hero quality
  • Entire image-to-rigged-character pipeline completed in under an hour, online only

AI compresses ideation and base-asset stages. Craft stages — clean topology, deformation rigs, signature animation — remain manual. Well short of the 50%+ vendors claim.
Key Finding
Hosted for Direction, Local for Volume
Use ChatGPT / Gemini for finalization and direction. Use local models (SDXL, FLUX, Trellis) for cost-controlled production volume. Studios doing all-hosted burn tokens.
Key Finding
AI Animation Claims Are Overstated
Auto-rigging and animation generation are not production-ready. Plan rigging and animation around manual craft, asset packs and mocap-cleanup tools.
Key Finding
Human Direction Is the Bottleneck
AI scales taste, it does not replace it. The hire that matters most is the art director who curates output — not the prompt engineer.
